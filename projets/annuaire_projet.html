Projet 1 - Annuaire Symfony

Contexte du projet

Le projet "Annuaire Symfony" consiste en la création d'une application web d'annuaire en ligne. Ce projet a été réalisé en trinôme et utilise le framework Symfony en combinaison avec Twig pour le rendu côté serveur (server-side rendering). L’objectif principal est de développer une plateforme permettant aux utilisateurs de créer, gérer et afficher leurs profils, tout en ayant un contrôle sur la visibilité de ces derniers.

Description du projet

L'application a été développée pour répondre à plusieurs besoins pratiques dans le cadre d'un annuaire numérique. Chaque utilisateur peut créer un profil avec des informations personnelles de base, modifier ces informations par la suite et choisir la visibilité de son profil. Les profils peuvent être publics (visibles) ou privés (masqués). Les utilisateurs peuvent également choisir de définir un code unique pour leur profil, ou opter pour un code généré automatiquement.

Fonctionnalités principales :

  • Création et gestion des profils utilisateurs : L’utilisateur peut créer un profil en renseignant son adresse e-mail, un mot de passe et une visibilité pour son profil. Un code unique peut être généré ou choisi par l’utilisateur lors de l’inscription. Le formulaire d’inscription est validé par JavaScript pour vérifier l’unicité du code en temps réel.
  • Modification du profil : Après l'inscription, l'utilisateur peut éditer son profil en ajoutant des informations supplémentaires telles que son numéro de téléphone, son adresse, ou ses profils sur les réseaux sociaux. Ces informations sont pré-remplies lors de la modification, simplifiant ainsi l’expérience utilisateur.
  • Visibilité et accès au profil : Le profil peut être visible publiquement, affiché sur la page d’accueil de l’annuaire, ou masqué, mais toujours accessible via l’URL directe du profil. L’administrateur a la possibilité de voir tous les profils, y compris ceux qui sont masqués.
  • Gestion des rôles utilisateurs : Le site distingue les utilisateurs classiques des administrateurs. Ces derniers ont la capacité de supprimer les comptes d'autres utilisateurs, sauf pour les administrateurs eux-mêmes. Une commande Symfony permet de créer un utilisateur depuis le terminal, avec la possibilité de définir son rôle (utilisateur classique ou administrateur).
  • Mode maintenance : Un paramètre dans le fichier services.yaml permet de mettre le site en mode maintenance. En mode maintenance, toutes les pages du site redirigent vers une page expliquant que l’application est actuellement indisponible.
  • Dernière modification et connexion : La dernière date de modification du profil ainsi que la dernière connexion de l'utilisateur sont affichées sur la page de profil. Ces dates sont mises à jour de manière automatique, ce qui permet de garantir que les informations restent à jour.
  • Exposition des données : Les informations des utilisateurs peuvent être consultées en format JSON, ce qui ouvre la voie à des applications externes ou des API.

Technologies utilisées :

  • Symfony 6 : Framework PHP permettant une gestion simplifiée des routes, de la base de données et des services.
  • Twig : Moteur de templates utilisé pour le rendu des pages HTML.
  • JavaScript natif : Utilisé pour les validations et l'interface utilisateur dynamique, comme la vérification en temps réel du code du profil.
  • Doctrine ORM : Pour la gestion des entités utilisateurs et des opérations sur la base de données.

Conclusion

Ce projet a permis de développer une application fonctionnelle d’annuaire en ligne, offrant aux utilisateurs un contrôle total sur leurs profils, et ce, avec une interface simple et claire. Il a également permis d'approfondir les connaissances sur Symfony, la gestion des utilisateurs et l'intégration d'outils pour gérer l'expérience utilisateur avec JavaScript natif.
